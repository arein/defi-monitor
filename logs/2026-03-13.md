### Headline

A user sold roughly $50M USDT for approximately $36K worth of AAVE through the Aave mobile UI routed via CoW Swap, accepting an onchain minimum-received amount that locked in ~99% slippage; Titan Builder extracted the MEV and the channel spent the day arguing whether the failure belongs to the protocol design or the UI.

### Incidents

- **Aave / CoW Swap $50M swap (other, high)**: The attacker / user signed a CoW order specifying a minimum-received amount that implied near-zero AAVE, and CoW's solvers executed to that floor. Stani Kulechov confirmed it was an Aave mobile UI flow with explicit warning-checkbox approval ([Stani](https://x.com/StaniKulechov/status/2032193345414664659)). Emilio (Aave) noted session replay shows the user acknowledged the warning. radu pointed out this is not slippage in the conventional sense: "cow gave the user a bad quote which the user (carelessly?) signed... minimum received you sign for is enforced at the contract level." Tom initially attributed to slippage misconfig, later updated to "signed a txn without an appropriate minimum." Meek argued all the way down that the protocol design is at fault for even routing such an order: `partial order filling... you can just give them $36k for $36k and return the rest is that hard?`. Damian pushed back: "slippage is the difference between what the trade is expected to be, and the trade that is going to execute on chain... the expected trade was just complete shit, and slippage is irrelevant in such cases." Tay's summary: UI should have failed closed on a 50M-for-36k trade, period. Titan Builder pocketed the MEV. zek (LlamaRisk) called out that CoW still routes >50% price-impact trades with only a yellow warning. CoW will refund their fees; Meefs posted the [CoW response](https://x.com/CoWSwap/status/2032265762916757614).

### Discussion

UI-vs-protocol debate dominated. Julie (likely Bankless Consulting) pushed the "UIs are products, not protocols" line that Odysseas echoed late: `A UI is a product, not a protocol... A product should never allow this outcome, plain and simple.` Pancake's 49%-slippage-block in 2021 resurfaced as a precedent. Meek promoted Asyncswap's partial-order-filling approach; Julie argued it's a UI problem not protocol ("broadcast your real intent"). iphelix marked the **3-year anniversary of the Euler hack and recovery**. Tay shared [innercitypress](https://x.com/innercitypress/status/2032483477547811030) as oblique context.

_Also mentioned: rashmor published [Git Fix Finder](https://x.com/rashmor_eth/status/2032286689654116840) turning git history into a vulnerability-fix corpus for auditors, David / Popeye tracking the attacker-commission address (`0xC542C2F197c4939154017c802B0583C596438380`) which matches Aave's fee receiver from the Rekt [Aave winter-of-discontent](https://rekt.news/aave-winter-of-discontent) piece, and Alex (GetRecon) teasing a function-chunking super-audit tool._
