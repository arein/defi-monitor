### Headline

Giddy.co lost $1.3M to a replay-signature exploit in GiddyVaultV3 where EIP-712 coverage skipped the `aggregator` / `fromToken` / `toToken` / `amount` fields; in parallel, Icalurus and Kuru's Devblixt spent the day publicly arguing whether Kuru's unbacked-collateral behavior on Monad is a critical protocol flaw or expected permissionless-DEX behavior, and Mozilla's AI-0day blog triggered Mythos-vs-Claude-4.6 benchmark talk.

### Incidents

- **Giddy.co $1.3M exploit (exploit, high)**: Icalurus posted the DefimonAlerts breakdown ([tweet](https://x.com/DefimonAlerts/status/2047334517535642024)): `GiddyVaultV3's _validateAuthorization() uses EIP-712 signatures that only cover the data bytes of each SwapInfo struct, but NOT the aggregator, fromToken, toToken, or amount fields.` Attacker replayed a valid signature with modified `SwapInfo`: `fromToken` set to the strategy's staked LP tokens, `aggregator` set to the attacker's contract, `toToken` a fake attacker-created token, `amount` = MAX_UINT256. Attack tx: [etherscan](https://etherscan.io/tx/0x5edb66a4c2ea55bba95d36d27713e3bb1c67c3c4199a8a1759e754c6f25482e5).
- **Kuru "lying-token" unbacked-collateral dispute (exploit, med)**: Icalurus published a [Kuru-PoC repo](https://github.com/DK27ss/Kuru-PoC) demonstrating that Kuru (on Monad) credits margin based solely on `transferFrom()` return value without verifying that tokens were actually received, allowing `LyingToken()` to inflate margin and drain real counterparty assets via `placeAndExecuteMarketBuy()` (traces at Monad block 69179576, lines 1493-1778). Kuru reclassified the report as medium + duplicate and Icalurus disputed publicly: "create pool with phantom liquidity... lying margin inflation... k manipulation... never mind, there are hundreds of other protocols on Defilama lmao." Devblixt (Kuru) countered: "standard bounty rejection tbh... a known behaviour since it is a permissionless protocol... The only way you can lose money in this is if you make a direct call to the market for a market order, knowing that you are trading an unknown and potentially malicious token." Damian cross-examined and concluded "if victim doesn't need to interact with a malicious token... still everyone gets wrecked, That is a critical vuln then." Icalurus: "victim only executes standard flow approveAndDeposit(), marketBuy(), withdraw() while the exploit is enabled by Kuru protocol already accepting invalid collateral." Mikko's nitpick: "this is only Ethereum problem and chains can standard around safe immutable tokens like Solana did." Net: Kuru maintains it's not a vuln, some researchers disagree, no funds drained but a standing disagreement.
- **Mozilla AI 0days (infrastructure, med)**: Fernando flagged the [Mozilla blog](https://blog.mozilla.org/en/privacy-security/ai-security-zero-day-vulnerabilities/). Michael: "Not bad. It is still far from that good for smart contracts." Dickson's headline: "jesus 22 with 4.6 to 271 with mythos." - **Self-propagating npm worm (infrastructure, high)**: Anatol linked [ThreatRoad writeup](https://threatroad.substack.com/p/a-self-propagating-npm-worm-is-actively) on an active npm self-propagation attack, extending the supply-chain wave from late March.
